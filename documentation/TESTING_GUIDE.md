# 🧪 Полное тестирование проекта локально

Полное руководство по запуску, разработке и тестированию Poetry Site на локальной машине.

## ✅ Предварительная проверка

Перед запуском убедитесь, что у вас установлено:

```bash
# Проверка Docker
docker --version
# Docker version 20.10.0+

# Проверка Docker Compose
docker-compose --version
# docker-compose version 1.29.0+
```

## 🚀 Быстрый старт (3 минуты)

### 1. Запустите проект

```bash
cd /Users/aleksandr/Local/poetry-site
docker-compose up
```

### 2. Откройте в браузере

```
http://localhost
```

### 3. Результат

Вы должны увидеть главную страницу с навигацией и переключателем языков (EN/RU).

## 🧬 Детальное тестирование всех функций

### Часть 1️⃣: Тестирование Frontend

#### 1.1 Многоязычность (i18n)

**Шаг 1: Переключение языка**
- Откройте `http://localhost`
- В правом верхнем углу найдите кнопки `EN | RU`
- Нажмите `RU` → интерфейс должен переключиться на русский
- Нажмите `EN` → интерфейс вернётся на английский

**Проверяемые элементы:**
- ✅ Заголовок сайта (`Words in the dark` ↔ `Слова в темноте`)
- ✅ Меню навигации (Home ↔ Главная, Poems ↔ Стихи, About ↔ Об авторе)
- ✅ Кнопки (Login ↔ Войти, Logout ↔ Выйти)
- ✅ Сообщения об ошибках

**Проверка сохранения:**
- Переключитесь на русский
- Обновите страницу (F5 / Cmd+R)
- Язык должен остаться русским ✅

#### 1.2 Навигация между страницами

**Главная страница (Home)**
- Откройте `http://localhost`
- Должны видеть: заголовок, подзаголовок, кнопку "Read the poems"
- Должна быть секция "Latest" (или "Новое" на русском)

**Страница Стихов (Poems)**
- Нажмите на "Poems" в меню
- URL должен измениться на `/poems`
- Должны видеть: список всех стихов (пока пуст или несколько тестовых)

**Страница об авторе (About)**
- Нажмите "About"
- URL должен измениться на `/about`
- Должны видеть: информацию об авторе

**Тестирование браузерной истории:**
- Нажмите "Back" в браузере
- Должно вернуться на предыдущую страницу

#### 1.3 Тестирование responsiveness

**На десктопе (1920x1080):**
- Все элементы видны правильно
- Переключатель языков видна в правом углу

**На мобиле (375x667):**
- Откройте DevTools (F12) → Toggle device toolbar
- Проверьте, что:
  - ✅ Меню адаптируется
  - ✅ Текст читаемый
  - ✅ Кнопки кликабельны

### Часть 2️⃣: Тестирование Backend API

#### 2.1 Проверка здоровья API

```bash
curl http://localhost/api/health
# Ответ: {"status":"ok"}
```

#### 2.2 API документация

Откройте в браузере:
- **Swagger UI**: `http://localhost/api/docs`
- **ReDoc**: `http://localhost/api/redoc`

Там видно все доступные эндпоинты.

#### 2.3 Получение стихов

```bash
curl http://localhost/api/poems
# Ответ: []  (пусто или список стихов)
```

#### 2.4 Получение информации об авторе

```bash
curl http://localhost/api/about
# Ответ: {"id":1,"name":"Famous poet","bio":"Welcome to my poetry corner.","photo_url":null,"updated_at":"..."}
```

### Часть 3️⃣: Тестирование администраторской панели

#### 3.1 Вход в админ-панель

1. Нажмите "Login" в правом верхнем углу
2. Введите учётные данные:
   - Username: `admin`
   - Password: `changeme123`
3. Нажмите "Login"
4. Должны увидеть сообщение об успешном входе

#### 3.2 Создание стихотворения

1. Откройте admin панель (`/admin`)
2. На вкладке "Poems" нажмите "+ New Poem"
3. Заполните форму:
   - **Title** (опционально): "My First Poem"
   - **Body** (обязательно): "This is my first poem"
   - **Tags** (опционально): "love, nature"
4. Нажмите "Publish"
5. Должны увидеть сообщение "Poem published"
6. Стихотворение должно появиться в списке

#### 3.3 Редактирование стихотворения

1. В списке администраторских стихов найдите только что созданное
2. Нажмите "Edit"
3. Измените текст или название
4. Нажмите "Save changes"
5. Должны увидеть "Poem updated"

#### 3.4 Удаление стихотворения

1. Нажмите "Delete" возле стихотворения
2. Подтвердите удаление
3. Стихотворение должно исчезнуть из списка

#### 3.5 Редактирование страницы об авторе

1. Перейдите на вкладку "About Page"
2. Заполните форму:
   - **Your Name**: "Лев Горев"
   - **Photo URL**: "https://example.com/photo.jpg"
   - **Bio**: "Я пишу стихи о любви и природе"
3. Нажмите "Save"
4. Должно сказать "About page saved"
5. Откройте `/about` и проверьте, что изменения применились

#### 3.6 Смена пароля

1. Перейдите на вкладку "Password"
2. Заполните форму:
   - **Current Password**: `changeme123`
   - **New Password**: `mynewpassword123`
   - **Confirm New**: `mynewpassword123`
3. Нажмите "Change Password"
4. Должно сказать "Password changed"
5. Выйдите из админ-панели ("Logout")
6. Попробуйте войти с новым паролем

### Часть 4️⃣: Тестирование комментариев

#### 4.1 Добавление комментария

1. Создайте и опубликуйте стихотворение (см. 3.2)
2. На главной странице или странице стихов откройте стихотворение
3. Внизу должна быть секция "Reflections" (или "Размышления" на русском)
4. Заполните форму:
   - **Name** (опционально): "John"
   - **Reflection**: "Beautiful poem!"
5. Нажмите "Post"
6. Комментарий должен появиться в списке

#### 4.2 Удаление комментария

1. В админ-панели перейдите к стихотворению
2. Должны видеть список комментариев
3. Нажмите "Delete" возле комментария
4. Подтвердите
5. Комментарий должен исчезнуть

### Часть 5️⃣: Тестирование фильтрации по тегам

#### 5.1 Фильтрация

1. Откройте страницу `/poems`
2. В списке тегов нажмите на один из тегов (например, "love")
3. Должны отобразиться только стихотворения с этим тегом
4. URL должен измениться на `/poems?tag=love` (или `/poems?tag=природа`)

#### 5.2 Очистка фильтра

1. Нажмите на "All" в списке тегов
2. Должны вернуться все стихотворения

### Часть 6️⃣: Тестирование многоязычности в админ-панели

1. Войдите в админ-панель
2. Переключитесь на русский язык (`RU`)
3. Проверьте, что все элементы админ-панели переведены:
   - Заголовки форм
   - Лейблы полей
   - Кнопки
   - Сообщения об ошибках

### Часть 7️⃣: Тестирование базы данных

#### 7.1 Проверка БД

```bash
# Войдите в контейнер backend
docker-compose exec backend bash

# Откройте SQLite
sqlite3 /data/poetry.db

# Проверьте таблицы
.tables
# Должны видеть: admin, about, poems, tags, poem_tags, comments

# Посмотрите данные
SELECT * FROM admin;
SELECT * FROM poems;
SELECT * FROM comments;

# Выйдите
.exit
```

#### 7.2 Проверка сохранения данных

1. Создайте стихотворение
2. Остановите контейнер: `docker-compose down`
3. Запустите снова: `docker-compose up`
4. Проверьте, что стихотворение всё ещё там

## 🐛 Отладка

### Просмотр логов

**Все логи:**
```bash
docker-compose logs
```

**Логи только backend:**
```bash
docker-compose logs backend
```

**Логи только nginx:**
```bash
docker-compose logs nginx
```

**Логи в реальном времени (tail):**
```bash
docker-compose logs -f
```

### Остановка контейнеров

```bash
# Остановить все
docker-compose down

# Остановить и удалить volumes (БД)
docker-compose down -v

# Пересоздать all контейнеры
docker-compose up --force-recreate
```

### Вход в контейнер

```bash
# Backend
docker-compose exec backend bash

# Nginx
docker-compose exec nginx bash

# Внутри контейнера можно запустить команды:
ls /app
cat /data/poetry.db (неработает, это бинарный файл)
```

## 🧼 Чистка и переустановка

### Полная переустановка

```bash
# Остановить всё
docker-compose down -v

# Удалить все образы проекта (опционально)
docker image rm poetry-site-backend poetry-site-nginx

# Запустить снова
docker-compose up --build
```

## 🔐 Тестирование безопасности

### 1. JWT токены

```bash
# После входа, откройте консоль браузера (F12)
localStorage.getItem('token')
# Должны видеть JWT токен
```

### 2. Попытка несанкционированного доступа

```bash
# Попробуйте создать стихотворение без токена
curl -X POST http://localhost/api/poems \
  -H "Content-Type: application/json" \
  -d '{"title":"test","body":"test"}'

# Ответ: 403 Forbidden
```

### 3. Если нет токена

```bash
# Должно вернуть 401
curl http://localhost/api/admin
# Ответ: {"detail":"Not authenticated"}
```

## 🌐 Тестирование CORS

# CORS должен быть разрешён для всех origin

## 📝 Команды для быстрого тестирования

Создайте файл `test.sh`:

```bash
#!/bin/bash

echo "🚀 Запуск тестов..."

# Проверка здоровья API
echo "Checking API health..."
curl http://localhost/api/health

echo ""
echo "Getting poems..."
curl http://localhost/api/poems

echo ""
echo "Getting about..."
curl http://localhost/api/about

echo ""
echo "✅ Тесты завершены!"
```

Запуск:
```bash
chmod +x test.sh
./test.sh
```
