# 📚 Полная документация исправления ошибки Docker Compose

## 🎯 Суммарная информация

### ✅ Что было исправлено

**Ошибка:**
```
failed to solve: failed to compute cache key
error: "/default.conf": not found
```

**Причина:**
Файл `/nginx/Dockerfile` использовал неправильные пути для копирования файлов при сборке Docker образа.

**Решение:**
Исправлены пути в `COPY` команд с использованием относительных путей от корня проекта.

**Файл, который был изменён:**
- `/nginx/Dockerfile`

## 🐛 Решение проблем

### Проблема 1: Ошибка при запуске
```bash
# Решение: полный перезапуск
docker-compose down -v
docker-compose up --build
```

### Проблема 2: Порт занят
```bash
# Измените в docker-compose.yml:
ports:
  - "8080:80"  # Вместо 80

# Откройте: http://localhost:8080
```

### Проблема 3: БД повреждена
```bash
# Удалите БД volume:
docker-compose down -v

# Пересоздавайте:
docker-compose up
```

**Полное руководство:** [SETUP_LOCAL.md](SETUP_LOCAL.md) → Раздел "Решение проблем"
